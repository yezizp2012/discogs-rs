# Security Policy

## Supported Versions

Security fixes are applied to the latest released version.

| Version | Supported |
| --- | --- |
| latest | :white_check_mark: |
| older  | :x: |

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Report privately via GitHub Security Advisories:
- Go to the repository's **Security** tab
- Click **Report a vulnerability**

If Security Advisories are unavailable, contact the maintainers directly and include:
- impact summary
- reproduction details
- affected versions
- potential mitigation

## Response Expectations

- Initial triage response: within 3 business days
- Status update cadence: at least once per week until resolution
- Fix + disclosure timeline: based on severity and exploitability
